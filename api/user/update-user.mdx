---
title: Update user
description: Update user information and settings
---

## Endpoint

```http
PUT /api/v1/users/{id}
```

Updates user information including name, email, push token, and notification preferences. All fields are optional - only include the fields you want to update.

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier (UUID) of the user to update
</ParamField>

## Request body

<ParamField body="name" type="string">
  Updated name of the user
</ParamField>

<ParamField body="email" type="string">
  Updated email address (must be valid email format)
</ParamField>

<ParamField body="push_token" type="string">
  Updated Firebase Cloud Messaging token for push notifications
</ParamField>

<ParamField body="preferences" type="object">
  Updated notification preferences
  
  <Expandable title="Preferences object">
    <ParamField body="email" type="boolean">
      Enable or disable email notifications
    </ParamField>
    
    <ParamField body="push" type="boolean">
      Enable or disable push notifications
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns the updated user object with all fields.

<ResponseField name="id" type="string">
  User's unique identifier (UUID)
</ResponseField>

<ResponseField name="name" type="string">
  Updated user name
</ResponseField>

<ResponseField name="email" type="string">
  Updated email address
</ResponseField>

<ResponseField name="push_token" type="string">
  Updated FCM token (or null if not set)
</ResponseField>

<ResponseField name="preferences" type="object">
  Updated notification preferences
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the update
</ResponseField>

## Example request

```bash cURL
curl -X PUT http://localhost:8001/api/v1/users/123e4567-e89b-12d3-a456-426614174000 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Updated",
    "push_token": "new_fcm_token_xyz789"
  }'
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:8001/api/v1/users/123e4567-e89b-12d3-a456-426614174000',
  {
    method: 'PUT',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'John Updated',
      push_token: 'new_fcm_token_xyz789'
    })
  }
);

const updatedUser = await response.json();
```

```python Python
import requests

user_id = '123e4567-e89b-12d3-a456-426614174000'
response = requests.put(
    f'http://localhost:8001/api/v1/users/{user_id}',
    json={
        'name': 'John Updated',
        'push_token': 'new_fcm_token_xyz789'
    }
)

updated_user = response.json()
```

## Error responses

<ResponseField name="404 Not Found" type="error">
  User with the specified ID does not exist
  
  ```json
  {
    "statusCode": 404,
    "message": "User not found"
  }
  ```
</ResponseField>

<ResponseField name="400 Bad Request" type="error">
  Invalid email format or malformed request
  
  ```json
  {
    "statusCode": 400,
    "message": "Validation failed",
    "errors": ["email must be a valid email"]
  }
  ```
</ResponseField>

## Use cases

- Update user profile information
- Change notification device token when user logs in on new device
- Modify contact information
- Combined with preferences update for comprehensive profile changes

<Note>
This endpoint updates the entire user object. Use the [Update user preferences](/api/user/update-preferences) endpoint if you only need to modify notification settings.
</Note>
